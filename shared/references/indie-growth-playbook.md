# Indie Growth Playbook

## Launch surfaces (use what fits the product)

| Surface | Best for | Effort |
|---------|---------|--------|
| Product Hunt | Consumer / prosumer / dev tools | High (1 day prep) |
| Hacker News | Dev tools, technical products, "Show HN" | Low (1 hour) |
| X / Twitter | Build-in-public, dev audience | Continuous |
| Reddit (subreddit-specific) | Niche communities | Medium (must read rules) |
| Indie Hackers | Indie tools, SaaS | Medium |
| LinkedIn | B2B, ops, sales tools | Medium |
| TikTok / YouTube | Consumer, visual products | High (content cadence) |

## Pre-launch checklist (1 week before)

- [ ] Landing page live with clear value prop
- [ ] Pricing page (even if just "Free during beta")
- [ ] Onboarding flow tested with 3 strangers
- [ ] Analytics installed and verified
- [ ] Feedback channel (email / Discord / form) live
- [ ] Status page or last-resort contact (email)
- [ ] Press kit / screenshots / 30-second demo video

## Post-launch (first 2 weeks)

- Reply to every comment in <2 hours during launch day
- Track 3 metrics: signups, activation %, paying conversion
- Ship 1 visible improvement per day from feedback
- Decide at day 14: iterate / kill / scale

## Feedback triage

| Feedback type | Action |
|--------------|--------|
| Bug | Fix today if blocks core flow, else log |
| Feature request | Ask "what problem are you solving?" before building |
| Pricing complaint | Log, do not adjust until 5+ similar |
| Praise | Save quote for marketing |
| Churn reason | Most important — interview the user |
